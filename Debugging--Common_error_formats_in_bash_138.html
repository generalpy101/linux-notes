Common error formats in bash


Basically, shell errors are formatted in the way shown below :

command: text or line where error occured: Error description

In our above example, ls is the command, cannot access 1234 is the place where error occured and No such... is error description. This is the preffered way to show errors according to bash convention.

Some of the common bash errors are :

Syntax errors where something is not written according to bash syntax.

Here we can see that error occured in line 5 which is syntax error. We can fix this by correcting the syntax. Shellcheck can also be used.

Errors when creating errors are common in bash scripting. Like in our above ls example, we were missing 1234. Since linux doesn't differentiates between files and dirs very much, error message shows that file or directory is not found. Always check if the path is correctly spelled and exists or is not subjected to any word splitting or globbing.

Another common error is when we are trying to create a file or dir and it already exists.

Always ensure that directory or file with same name doesn't exists.

File permissions can also cause errors like creating files where we are not allowed to create.

Also if commands are not found, it can give errors. Ensure that your script is in path and has appropriate execution permissions.
