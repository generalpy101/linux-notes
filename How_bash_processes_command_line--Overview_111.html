Overview


Bash follows 5 step process execute a command given to it :

1. Tokenisation : Tokenisation is the process dividing command into tokens. Token is a sequence of characters which is smallest single unit of the command. Tokens are divided according to metacharacters.

There are 2 types of tokens in bash :

<i>. Words : A word is a token that doesn't contain any unquoted metacharacter.

<ii>. Operator : An operators is a token that contain atlease 1 unquoted metacharacter.

2. Command identification : After tokenisation, command identification is done and command is divided into 2 categories :

<i>. Simple commands are the commands which are just a bunch of individual words and are terminated by control operators. We've been using simple commands till now which are command names followed by its arguments.

<ii>. Complex commands gives bash its programming constructs like if,else , while loop etc.

3. Expansions : If any type of expansions are present, commands are expanded into it.

4. Quote removal : Quotes can be used to control the flow of commands. In this step quotes are removed and its components are evaluated accordingly.

5. Redirection : More on this later.
