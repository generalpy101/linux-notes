Quoting


Commands are just a bunch of character strings.

Quoting plays an important role in bash. Quoting is used to remove special meaning from commands.

There are 3 types of quoting in bash :

1. Using backslash(\) removes special meaning from the character immediately followed by it.

2. Using single quotes('') removes special meaning from anything enclosed in them.

3. Using double quotes("") removes special meaning from anything enclosed in them except $ and ``.

Let's look at an example :

We want to store Windows path which contain backslashes which are itself quoting characters.

Path is : C:\Users\user1\Desktop and C:\Users\$USER\Desktop(for any user).

We saw that how we can use quotes to manipulate our string. In first case, backslash itself is quoting character hence it was resolved, in this case to nothing as character following it have no special meaning. Then we used single quotes to get our desired outpu tand remove special backslashes. After that we use single quotes on a variable but it was nullified so we used double quotes to get our desired result.
