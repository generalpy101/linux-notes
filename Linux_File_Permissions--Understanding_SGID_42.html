Understanding SGID


We saw that we have some special file permissions for files. SGID is also a special file permission that applies to groups.

We generally use SGID on directories.

When SGID is set for a directory, all contents in that directory have the same group as the parent directory.

To set SGID, there are 2 ways :

chmod g+s file

chmod 2xxx file

SGID is usually used to create shared directories.

Using above commands we created a directory with group as programmers. Now generalpy created a file inside the dir and the directory had group same as generalpy. When james, who is in same programmers group want to edit that file created by generalpy(assume that both work in same office), he cannot do that. We have to set SGID in this case.

Now we changed user directory to SGID directory. See the difference between fil a and b. Group of file b is programmers inspite of being created by generalpy.
