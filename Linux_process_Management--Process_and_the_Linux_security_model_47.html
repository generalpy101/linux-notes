Process and the Linux security model


Running instance of any program is know as a process and it runs in its own memory space.

Each process is an active entity as opposed to programs which are passive entities.

As stated earlier, there are 2 type of commands in linux ,the executable binaries and builtin shell commands. Only executables spawn a process every time they are executed.

Each process in Linux has a PID(process ID), user , group and priority. For a binary, until the suid is set, command is run as the current user which may or may not have the root priivileges. This practice makes linux very secure and it is not recommended to run any command with root privileges until it is required.

There are different types of processes :

Parent

Child

Daemon

Zombie(defunct)

Orphan

When a linux system boots, a process starts which is know as init or systemd. It is the first process in the system and has the PID of 1. All other processes fork from this process making other process its child.

A parent process is a special process which invokes multiple processes. The processes invoked by that parent process are known as child process.

When a process ends, its resources like memory space are freed automatically but in some cases that cannot be done. These type of process are known as zombie or defunct process.

An orphan process is opposite of zombie process. Its a process whose parent process terminated before it. In this case os sends a hangup signal to that process and ultimately terminates it.

A daemon process is a special process which runs in background and is run with root privleges to provide some special functionality.

Note that task ans process are same terms.
