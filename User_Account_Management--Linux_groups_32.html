Linux groups


A group is a collection of users that can be used to provide combined permissions and ownerships.

There are 2 types of groups a user can belong to. Primary group which is the group whose id is stored in /etc/passwd. This group usually has same name as user's and is assigned to user when creating account. Secondary group is the group in which user is added after creation of user account.

Each user can only belong to only primary group but n number of secondary groups.

/etc/group stores the name of the group mapped to corresponding id.

Let's see group of our current user.

We can see that primary group id of user generalpy is 1000 which maps to group named generalpy.

Last field in /etc/group file shows all users which belong to that group, separated by commas.

Here we can see all groups to which user generalpy belongs.

There is dedicated groups command to do this group searching stuff.

If no argument is provided to groups command, it lists all groups to which current user belongs to and first group is primary group.

To find groups of any user, provide that user's name to groups command.

Another important command to know groups is id. It woks same as groups in terms of arguments.

Id shows userid group id and groups of user(current user in this case).
