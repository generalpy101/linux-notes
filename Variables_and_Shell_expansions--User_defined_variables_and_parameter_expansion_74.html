User defined variables and parameter expansion


According to bash manual, a parameter is an entity that can store data.

There are 3 types of parameters :

1. Variables

2. Positional Parameters

3. Special Parameters

Variable is a parameter whose parameter can be manually changed.

There are 2 types of variables : User defined and Shell variables.

Shell variables are written in all caps and user defined variables are written in all lower case(just a convention).

To define a variable we write 

variable_name=value

There shouldn't be any space between variable name, = and variable value as if there is a space, bash will think that variable name is a command.

To extract value of an variable we use ${variable_name}. This syntax is known as shell expansion.

#!/bin/bash

name="generalpy"

echo "Hello ${name}"

exit 0

Variables are very useful in a bash script as we can write a value once and if we need to change something we can directly change the variable value instead of changing all the occurences.

If a variable doesn't exists the variable expansion statement evaluates to nothing.
